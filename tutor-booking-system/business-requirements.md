# Бизнес-требования

## 1. Контекст

Компания предоставляет услуги репететирства по раличным предметам.

В настоящее время создание пользователей, ведение расписания, запись учеников на уроки, отмена уроков происходит вручную через месенджер. 

## 2. Проблема

Сейчас запись на занятия происходит вручную через мессенджеры. Репетитор тратит время на переписку, проверку расписания и ведение записей в таблицах.

Из-за ручного процесса возможны ошибки:

-двойная запись на одно время;
-потеря информации о занятии;
-забытые отмены;
-длительное согласование времени.

## 3. Цель проекта

- автоматизировать процесс записи учеников на занятия;
- снизить количество ручных действий репетитора;
- уменьшение ошибок;
- упрощение управления расписанием.

## 4. Основные возможности
### Ученик

- регистрация и вход в личный кабинет;
- просмотр доступных временных слотов;
- запись на занятие;
- отмена занятия;
- просмотр истории занятий.

### Репетитор

- создание свободных временных слотов;
- просмотр расписания;
- отмена занятия;
- просмотр списка учеников.

### Администратор

- управление пользователями;
- просмотр всех занятий;
- блокировка пользователей при необходимости.

## 5. Бизнес-требования

- BR-001. Система должна уменьшить количество ручных действий при записи учеников на занятия.
- BR-002. Система должна снизить вероятность ошибок при ведении расписания занятий.
- BR-003. Система должна увеличить количество обслуживаемых учеников без расширения штата.
- BR-004. Система должна сократить время согласования записи ученика на занятие.
- BR-005. Система должна упростить управление расписанием занятий для репетитора.
- BR-006. Система должна обеспечивать отображение актуального состояния расписания для ученика и репетитора.
