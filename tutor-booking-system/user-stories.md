# User Stories и Use Cases

### US-001 / UC-001. Запись на занятие

**User Story:**
Как ученик, я хочу записаться на свободный слот занятия, чтобы выбрать удобное время без переписки с репетитором.

**Связанные функциональные требования:**
FR-004, FR-005, FR-022, FR-024, FR-028, FR-029, FR-030, FR-031, FR-036

**Актор:**
Ученик

**Предусловия:**

* Ученик авторизован в системе.
* В системе есть свободные временные слоты, созданные репетитором.

**Критерии приёмки:**

* Ученик может просмотреть доступные свободные слоты.
* Ученик может записаться только на свободный слот.
* Система не позволяет записаться на прошедшее время.
* Система не позволяет записаться на занятый слот.
* После записи слот становится недоступен для других учеников.
* Ученик получает подтверждение записи.
* Репетитор получает уведомление о новой записи.

**Основной сценарий:**

| Шаг | Актор   | Действие                                                                    |
| --- | ------- | --------------------------------------------------------------------------- |
| 1   | Ученик  | Открывает экранную форму с расписанием репетитора.                          |
| 2   | Система | Отображает список свободных временных слотов.                               |
| 3   | Ученик  | Выбирает подходящий свободный слот.                                         |
| 4   | Система | Отображает информацию о выбранном слоте: дату, время и репетитора.          |
| 5   | Ученик  | Подтверждает запись на занятие.                                             |
| 6   | Система | Проверяет, что выбранный слот свободен и не относится к прошедшему времени. |
| 7   | Система | Создаёт запись ученика на занятие.                                          |
| 8   | Система | Обновляет статус временного слота на «занят».                               |
| 9   | Система | Отправляет ученику подтверждение записи.                                    |
| 10  | Система | Отправляет репетитору уведомление о новой записи.                           |

**Альтернативные сценарии:**

* Если выбранный слот уже занят, система не создаёт запись и показывает сообщение об ошибке.
* Если выбранный слот относится к прошедшему времени, система не позволяет создать запись.
* Если при создании записи произошла ошибка, система не показывает сообщение об успешной записи.

### US-002 / UC-002. Отмена записи на занятие учеником

**User Story:**
Как ученик, я хочу отменить свою запись на занятие, чтобы освободить время, если я не могу прийти на занятие.

**Связанные функциональные требования:**
FR-003, FR-006, FR-023, FR-025, FR-032, FR-034, FR-037

**Актор:**
Ученик

**Предусловия:**

* Ученик авторизован в системе.
* У ученика есть активная запись на занятие.

**Критерии приёмки:**

* Ученик может отменить только свою запись на занятие.
* Система запрашивает подтверждение перед отменой записи.
* Система не позволяет отменить запись, если время занятия уже прошло.
* После отмены записи слот становится свободным.
* Ученик получает подтверждение отмены записи.
* Репетитор получает уведомление об отмене записи учеником.
* Информация об отменённой записи сохраняется в истории занятий.

**Основной сценарий:**

| Шаг | Актор   | Действие                                                     |
| --- | ------- | ------------------------------------------------------------ |
| 1   | Ученик  | Открывает экранную форму с расписанием своих занятий.        |
| 2   | Система | Отображает расписание занятий ученика.                       |
| 3   | Ученик  | Выбирает нужное занятие и нажимает кнопку «Отменить запись». |
| 4   | Система | Запрашивает подтверждение отмены записи.                     |
| 5   | Ученик  | Подтверждает отмену записи.                                  |
| 6   | Система | Проверяет, что время занятия ещё не прошло.                  |
| 7   | Система | Отменяет запись ученика на занятие.                          |
| 8   | Система | Обновляет статус временного слота на «свободен».             |
| 9   | Система | Отправляет ученику подтверждение отмены записи.              |
| 10  | Система | Отправляет репетитору уведомление об отмене записи учеником. |

**Альтернативные сценарии:**

* Если ученик не подтверждает отмену, система не отменяет запись.
* Если время занятия уже прошло, система не позволяет отменить запись и показывает сообщение об ошибке.
* Если при отмене записи произошла ошибка, система не меняет статус записи и показывает сообщение об ошибке.

### US-004 / UC-004. Создание свободного слота репетитором

**User Story:**
Как репетитор, я хочу создавать свободные временные слоты в своём расписании, чтобы ученики могли записываться на занятия.

**Связанные функциональные требования:**
FR-009, FR-010

**Актор:**
Репетитор

**Предусловия:**

* Репетитор авторизован в системе.

**Критерии приёмки:**

* Репетитор может открыть экранную форму со своим расписанием.
* Репетитор может создать свободный временной слот.
* Новый слот отображается в расписании репетитора.
* Новый слот доступен ученикам для записи.

**Основной сценарий:**

| Шаг | Актор     | Действие                                                                                             |
| --- | --------- | ---------------------------------------------------------------------------------------------------- |
| 1   | Репетитор | Открывает экранную форму со своим расписанием.                                                       |
| 2   | Система   | Отображает расписание репетитора.                                                                    |
| 3   | Репетитор | Нажимает кнопку «Добавить» в нужной временной ячейке расписания.                                     |
| 4   | Система   | Открывает форму создания свободного слота.                                                           |
| 5   | Репетитор | Указывает временной интервал, при необходимости добавляет комментарий и нажимает кнопку «Сохранить». |
| 6   | Система   | Сохраняет новый свободный временной слот в расписании репетитора.                                    |
| 7   | Система   | Показывает уведомление об успешном создании слота.                                                   |

**Альтернативные сценарии:**

* Если при загрузке расписания произошла ошибка, система показывает сообщение об ошибке.
* Если данные слота заполнены некорректно, система показывает сообщение об ошибке и не сохраняет слот.
* Если при сохранении слота произошла ошибка, система не создаёт слот и показывает сообщение об ошибке.

### US-008 / UC-008. Блокировка пользователя

**User Story:**
Как администратор, я хочу заблокировать учётную запись пользователя, чтобы ограничить доступ пользователя к системе без удаления его данных.

**Связанные функциональные требования:**
FR-016, FR-018, FR-027

**Актор:**
Администратор

**Предусловия:**

* Администратор авторизован в системе.
* Администратор открыл карточку нужного пользователя.
* Учётная запись пользователя не заблокирована.

**Критерии приёмки:**

* Администратор может заблокировать учётную запись пользователя.
* После блокировки пользователь не может войти в систему.
* Данные пользователя и история его занятий сохраняются в системе.
* В карточке пользователя отображается статус «заблокирован».

**Основной сценарий:**

| Шаг | Актор         | Действие                                                     |
| --- | ------------- | ------------------------------------------------------------ |
| 1   | Администратор | Нажимает кнопку «Заблокировать пользователя».                |
| 2   | Система       | Запрашивает подтверждение блокировки пользователя.           |
| 3   | Администратор | Подтверждает блокировку пользователя.                        |
| 4   | Система       | Меняет статус учётной записи пользователя на «заблокирован». |
| 5   | Система       | Сохраняет изменения в карточке пользователя.                 |
| 6   | Система       | Показывает уведомление об успешной блокировке.               |

**Альтернативные сценарии:**

* Если администратор не подтверждает блокировку, система не меняет статус пользователя.
* Если пользователь уже заблокирован, система не выполняет повторную блокировку и отображает текущий статус пользователя.
* Если при блокировке произошла ошибка, система не меняет статус пользователя и показывает сообщение об ошибке.

* После разблокировки пользователь может войти в систему.
* В карточке пользователя отображается статус «активен».
* Данные пользователя и история его занятий сохраняются.

**Основной сценарий:**

| Шаг | Актор         | Действие                                                |
| --- | ------------- | ------------------------------------------------------- |
| 1   | Администратор | Нажимает кнопку «Разблокировать пользователя».          |
| 2   | Система       | Запрашивает подтверждение разблокировки пользователя.   |
| 3   | Администратор | Подтверждает разблокировку пользователя.                |
| 4   | Система       | Меняет статус учётной записи пользователя на «активен». |
| 5   | Система       | Сохраняет изменения в карточке пользователя.            |
| 6   | Система       | Показывает уведомление об успешной разблокировке.       |

**Альтернативные сценарии:**

* Если администратор не подтверждает разблокировку, система не меняет статус пользователя.
* Если пользователь уже активен, система не выполняет повторную разблокировку и отображает текущий статус пользователя.
* Если при разблокировке произошла ошибка, система не меняет статус пользователя и показывает сообщение об ошибке.
