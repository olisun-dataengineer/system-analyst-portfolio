# Бизнес-требования

## 1. Общая информация

Компания предоставляет VPN-доступ по подписке.

В настоящее время создание пользователей, выдача конфигураций и продление подписок выполняются администраторами вручную.

Рост числа пользователей приводит к увеличению нагрузки на администраторов и повышает риск ошибок.

## 2. Цель проекта

Автоматизировать процессы управления VPN-подписками.

## 3. Бизнес-проблемы

- Ручное создание пользователей.
- Ручная выдача конфигураций.
- Ручное отключение пользователей после окончания подписки.
- Отсутствие уведомлений о скором окончании подписки.

## 4. Бизнес-цели

- Сократить время обслуживания клиентов.
- Снизить количество ручных операций.
- Снизить вероятность ошибок администраторов.
- Увеличить количество обслуживаемых пользователей без расширения штата.

## 5. Критерии успеха

- Не менее 90% операций выполняются автоматически.
- Пользователь может самостоятельно продлить подписку.
- Пользователь получает уведомление об окончании подписки.
- Администратор может управлять пользователями через веб-интерфейс.
